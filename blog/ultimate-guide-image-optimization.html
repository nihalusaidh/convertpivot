The Ultimate Guide to Image Optimization for Web

July 13, 2026

Here is a number that surprised me: images make up more than half the weight of the average web page. I have seen a single hero image bigger than the entire HTML, CSS, and JavaScript of the site around it. If you want your site to load fast — and Google definitely wants that — optimizing images is the easiest win you will ever get.

This guide covers everything I have learned about image optimization: picking the right format, compression tricks that actually work, responsive images, and how all of this ties into Core Web Vitals. Whether you are a developer, a designer, or just someone running a blog, these strategies will make your images load faster. For the full format showdown, see our HEIC vs JPEG vs WebP vs AVIF guide.

1. Choose the Right Image Format

The first and most important decision is choosing the right format for each image. Modern formats offer dramatically better compression than legacy formats, but browser support varies.

JPEG: Best for photographs and complex images. Universal browser support (100%). Lossy compression only. No transparency or animation. Use JPEG as a fallback format for maximum compatibility.

PNG: Best for graphics with sharp edges, text, logos, and screenshots. Lossless compression with transparency support. Larger file sizes than JPEG for photographic content. Use PNG when you need pixel-perfect reproduction or alpha transparency.

WebP: Best for modern web performance. Supports both lossy and lossless compression, transparency, and animation. Approximately 30% smaller than equivalent JPEG/PNG. Browser support at ~98%. WebP is the recommended format for most web images today.

AVIF: Best for maximum compression and future-proofing. Supports lossy and lossless, HDR, 12-bit color, and transparency. Approximately 50% smaller than JPEG. Browser support at ~92% and growing. Use AVIF with JPEG fallback for forward-looking optimization.

HEIC: Apple's format primarily for iPhone photography. Not recommended for web delivery due to limited browser support (Safari only). Convert HEIC to WebP or JPEG for web use using the HEIC to JPG converter.

2. Compression Techniques

Compression reduces file size by removing redundant or imperceptible image data. There are two approaches:

Lossy compression discards some image data to achieve smaller file sizes. The key is finding the sweet spot where quality loss is imperceptible but file size savings are significant. For JPEG, quality 80-85% is typically visually lossless. For WebP, quality 75-80% provides excellent results. For AVIF, quality 50-60% often matches JPEG quality 85% visually.

Lossless compression reduces file size without any quality loss by optimizing how data is stored. Tools like PNGGauntlet, OptiPNG, and JPEGoptim remove metadata (EXIF, GPS, color profiles) that is not needed for web display. Lossless savings are typically 5-25% compared to 50-80% with lossy compression.

For best results, use a combination: apply lossy compression to reduce the bulk of file size, then run lossless optimization to squeeze out remaining savings without additional quality loss.

3. Responsive Images with srcset and sizes

Serving the same large image to every device is wasteful. A 2000px-wide hero image is overkill for a 375px phone screen. The srcset and sizes attributes let you serve appropriately sized images based on the user's viewport.

Example: <img src="hero-800.jpg" srcset="hero-400.jpg 400w, hero-800.jpg 800w, hero-1200.jpg 1200w, hero-2000.jpg 2000w" sizes="(max-width: 600px) 100vw, (max-width: 1200px) 80vw, 1200px" alt="Hero image">. The browser automatically selects the most appropriate image from the srcset based on the current viewport width and pixel density.

The picture element adds format-based selection: <picture><source srcset="hero.avif" type="image/avif"><source srcset="hero.webp" type="image/webp"><img src="hero.jpg" alt="Hero"></picture>. Browsers that support AVIF will download the smaller AVIF file; others will fall back to WebP or JPEG. This is the gold standard for cross-browser image optimization.

4. Lazy Loading

Lazy loading defers loading off-screen images until the user scrolls near them. This dramatically reduces initial page weight and speeds up the first paint. Native lazy loading is built into modern browsers using the loading="lazy" attribute on img and iframe elements.

Implementation is simple: <img src="photo.jpg" loading="lazy" alt="Photo">. The browser handles everything — it loads the image when it is about to enter the viewport. For older browsers, JavaScript fallbacks using Intersection Observer API provide equivalent functionality.

Do not lazy load above-the-fold images (the hero image and initial content). These should load immediately for the best user experience. Use lazy loading for images below the initial viewport, gallery thumbnails, and content further down the page.

5. Image CDN and Automation

For production websites, image CDNs automate most optimization tasks. Services like Cloudinary, Imgix, and Cloudflare Images handle format selection (serving WebP/AVIF automatically), resizing, compression, and global delivery through CDN networks.

Image CDNs work by transforming images on-the-fly through URL parameters. A request like https://cdn.example.com/image.jpg?w=800&q=80&f=auto tells the CDN to resize to 800px width, apply 80% quality, and serve the best format (auto detects browser support). This eliminates the need to create and store multiple versions of each image.

For smaller sites without a CDN, build tools like ImageOptim, Squoosh, and Sharp (Node.js library) can automate optimization during your build process. These tools compress images and generate multiple sizes at build time, which are then deployed as static assets.

6. Image Optimization and Core Web Vitals

Google's Core Web Vitals directly measure user experience, and image optimization is the most impactful change you can make. Largest Contentful Paint (LCP) measures loading performance — images are the most common LCP element. Cumulative Layout Shift (CLS) measures visual stability — images without explicit dimensions are a leading cause of layout shifts.

To optimize LCP: Preload your hero image with <link rel="preload" as="image" href="hero.webp"> in the head. Serve appropriately sized images (don't make mobile users download desktop-sized images). Use modern formats like WebP or AVIF. Compress aggressively — a well-optimized hero image should be under 200 KB.

To prevent CLS: Always specify width and height attributes on your images. This reserves the correct space before the image loads. Combined with aspect-ratio CSS property, you can ensure images load without causing layout shifts even before they are fully downloaded.

Frequently Asked Questions

  1. What is the best image format for web performance? WebP is currently the best, offering 30% smaller files than JPEG with equivalent quality and 98% browser support. AVIF provides 50% smaller files than JPEG with 92% browser support. Use the picture element to serve modern formats with JPEG/PNG fallback.
  2. How does image optimization affect Core Web Vitals? Image optimization directly impacts LCP (Largest Contentful Paint). Unoptimized images are the most common cause of poor LCP scores. Proper optimization with appropriate dimensions, modern formats, and lazy loading can reduce LCP by 30-50%.
  3. What is the ideal image size for web use? There is no single ideal size — it depends on the display context. A hero image might need 1920-2560px wide for large screens. Thumbnails need only 150-300px. Serve appropriately sized images using srcset and sizes attributes. Aim for images under 100-200 KB for most content images, under 500 KB for hero images.
  4. Should I use JPEG or PNG for web images? Use JPEG for photographs and complex images with many colors. Use PNG for graphics with sharp edges, text, logos, icons, and images needing transparency. For best performance, consider converting both to WebP or AVIF with appropriate fallbacks.
  5. What is lazy loading and how does it help? Lazy loading delays loading off-screen images until the user scrolls near them. It improves initial page load time and reduces bandwidth usage. Native lazy loading with loading="lazy" is supported in all modern browsers.
