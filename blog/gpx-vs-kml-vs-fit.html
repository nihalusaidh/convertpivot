GPX vs KML vs FIT — GPS File Formats Compared

July 13, 2026

GPS data comes in many formats, and knowing the difference between GPX vs KML vs FIT is essential for anyone who works with mapping, outdoor navigation, or fitness tracking. Each format was designed for a different purpose: GPX for GPS devices, KML for Google Earth, and FIT for Garmin fitness watches. This guide breaks down each format so you can choose the right one and convert between them when needed.

GPX — The GPS Universal Standard

GPX (GPS Exchange Format) is the most widely adopted open standard for exchanging GPS data. It is an XML-based format that stores three types of data: waypoints (individual points of interest), tracks (recorded paths with timestamps), and routes (ordered lists of waypoints for navigation). GPX was developed by Topografix and is supported by virtually every GPS device manufacturer, including Garmin, Magellan, and Lowrance. It is also the primary format used by outdoor platforms like Strava, AllTrails, and Komoot. Because GPX is XML, files can be large, but the format is extremely well-documented and easy to work with programmatically.

KML — Google Earth's Visualization Format

KML (Keyhole Markup Language) was originally developed by Keyhole Inc. (later acquired by Google) for use with Google Earth. While GPX focuses on raw GPS data, KML is designed for rich geographic visualization. KML supports custom styles, colored lines and polygons, 3D models, ground overlays, and interactive pop-up descriptions with embedded HTML and images. This makes KML the best format for creating visually compelling maps for presentations, travel guides, and geographic analysis. KMZ files are compressed versions of KML (similar to ZIP) that are much smaller for sharing.

FIT — Garmin's Fitness Format

FIT (Flexible and Interoperable Data Transfer) is a binary data format developed by Garmin for fitness and health devices. Unlike the text-based GPX and KML, FIT uses a highly optimized binary structure that packs multiple data streams into a single compact file. A FIT file from a run or bike ride contains GPS coordinates, heart rate, cadence, speed, power output, temperature, elevation, and lap summaries — all in one file that is often smaller than the GPX track alone. FIT supports over 200 message types covering activities, workouts, monitoring data, device settings, and more. It is the native format for Garmin watches, Edge cycling computers, and Garmin Connect.

Which GPS Format Should You Use?

Use GPX for sharing routes with other GPS users, uploading to Strava or AllTrails, loading waypoints into a handheld GPS device, and any scenario requiring broad compatibility across GPS software and devices.

Use KML for creating maps in Google Earth, sharing geographic data with non-GPS users, building visual presentations with colored trails and markers, and integrating with GIS software like QGIS.

Use FIT for analyzing fitness activities from Garmin devices, exporting full workout data including heart rate and power, uploading to training platforms like TrainingPeaks or intervals.icu, and archiving workout data in the most space-efficient format.

Frequently Asked Questions

GPX (GPS Exchange Format) is an open XML-based standard for storing GPS data including waypoints, tracks, and routes. It was developed by Topografix and is the most widely supported format for exchanging GPS data between devices and applications. GPX files store latitude, longitude, elevation, and timestamp data for each point. Most GPS devices, including Garmin handheld units, support GPX natively. It is also supported by Strava, AllTrails, OpenStreetMap, and most mapping software.
KML (Keyhole Markup Language) is an XML-based format developed by Google for use with Google Earth and Google Maps. Unlike GPX which is purely a GPS data format, KML is a geographic visualization format that supports rich styling, 3D models, overlays, icons, and balloon descriptions. KML files can display colored trails, marked waypoints with custom icons, and even embedded HTML content. It is the best format for creating rich, visually annotated maps for sharing and presentation purposes.
FIT (Flexible and Interoperable Data Transfer) is a binary data format developed by Garmin for fitness and health devices. Unlike GPX and KML which are text-based XML formats, FIT uses a compact binary structure that stores data efficiently. A FIT file can contain GPS coordinates, heart rate, cadence, power, temperature, elevation, and lap information all in a single file. FIT files are significantly smaller than equivalent GPX or KML files, making them ideal for Garmin watches, cycling computers, and fitness trackers.
