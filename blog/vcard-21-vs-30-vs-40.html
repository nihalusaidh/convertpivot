vCard 2.1 vs 3.0 vs 4.0 — Which Version to Use

July 13, 2026

vCard, also known as VCF (Virtual Contact File), is the universal standard for exchanging contact information. However, not all vCards are created equal. The format has evolved through three major versions: 2.1, 3.0, and 4.0. Choosing the right version determines whether your contacts import correctly with special characters, photos, and custom fields. This vcard version comparison explains exactly what changed and which version you should use.

If you have ever tried importing a contact file from an iPhone into Outlook and found garbled names or missing photos, version mismatch is likely the culprit. Understanding the differences between vCard 2.1, 3.0, and 4.0 saves you from data loss during contact migration.

vCard 2.1 — The Legacy Standard

vCard 2.1 was published in 1996 by the Internet Mail Consortium (IMC) and became the baseline for digital business cards. It uses quoted-printable encoding for non-ASCII characters, which means accented letters, Chinese characters, and emoji must be encoded as escaped ASCII sequences. This encoding bloats file sizes and causes display issues in modern applications.

vCard 2.1 supports basic fields: name, phone, email, address, organization, and a single photo. It does not support structured name components (given name, middle name, family name are stored in a single NAME field). It also lacks support for multiple email addresses or phone numbers with distinct types (work, home, mobile).

Microsoft Outlook still uses vCard 2.1 as its primary exchange format, making it necessary for compatibility with legacy business systems and some CRM platforms.

vCard 3.0 — The Modern Standard

vCard 3.0, published in 2002 by the IETF as RFC 2426, was a major overhaul. It adopted UTF-8 as the default character encoding, eliminating the garbled text problem for non-English names. It introduced structured name properties (N, FN, GIVEN, MIDDLE, FAMILY), multiple phone and email entries with TYPE parameters, and support for inline BASE64-encoded photos.

vCard 3.0 is the most widely supported version today. Apple iPhone exports vCard 3.0 by default. Google Contacts uses vCard 3.0 for both import and export. Android devices generate vCard 3.0 files when sharing contacts. Most CRM systems including Salesforce, HubSpot, and Zoho support vCard 3.0 as the standard import format.

If you are unsure which version to use, choose vCard 3.0. It offers the best balance of feature support and compatibility across all major platforms.

vCard 4.0 — The Future Standard

vCard 4.0, published in 2011 as RFC 6350, is the most feature-rich version. It adds support for XML representations (xCard), JSON representations (jCard), KIND property for grouping contacts (individual, organization, group, location), and extended date/time formats including partial dates and durations.

vCard 4.0 also introduces PHOTO as a URI property, allowing photos to be referenced by URL rather than embedded as data. This dramatically reduces file sizes for contact lists with high-resolution images. Custom fields are fully supported through the extension mechanism, and social media profile links can be stored in a standardized way.

However, vCard 4.0 adoption lags behind vCard 3.0. Outlook has limited import support. Some CRM systems still reject vCard 4.0 files. For maximum compatibility today, stick with vCard 3.0 unless you specifically need vCard 4.0 features.

Which Version Should You Use?

Use vCard 2.1 if: You need compatibility with Outlook 2003-2010, legacy CRM systems, or certain enterprise contact databases. Also use 2.1 if you are exchanging contacts with older BlackBerry or Symbian devices.

Use vCard 3.0 if: You are migrating contacts between modern smartphones (iPhone, Android), importing into Google Contacts or iCloud, or exchanging contacts with contemporary CRM software. vCard 3.0 is the safe default for almost every scenario.

Use vCard 4.0 if: You need advanced features like group contacts, XML/JSON representations, or URI-based photos. Use 4.0 for modern contact management systems that explicitly advertise vCard 4.0 support.

Frequently Asked Questions

  1. What is a vCard file? A vCard (VCF) file is a standard format for electronic business cards, containing contact information like name, phone, email, address, photo, and custom fields for exchange between devices and applications.
  2. Which vCard version does iPhone use? Apple's iOS uses vCard 3.0 as the default export format. iPhones also import vCard 2.1 and can read many vCard 4.0 files.
  3. Can Outlook read vCard 4.0? Outlook has limited vCard 4.0 support. For best compatibility with Outlook, use vCard 2.1 or 3.0.
  4. How do I convert between vCard versions? Use a converter tool like VCF to CSV followed by CSV to VCF to re-export in a different version. Google Contacts also converts versions automatically on export.
  5. What is the difference between vCard 2.1 and 3.0? vCard 3.0 uses UTF-8 instead of quoted-printable encoding, supports structured name components, multiple phone/email entries per contact, and inline BASE64 photos. vCard 2.1 is limited to basic single-instance fields.
